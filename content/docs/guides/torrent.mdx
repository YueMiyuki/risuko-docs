Risuko includes a full BitTorrent client powered by [`risuko-bt`](https://github.com/YueMiyuki/Risuko/tree/main/src-tauri/risuko-bt), an in-tree BitTorrent v1 engine.

## Magnet Links

### CLI

```bash
risuko download "magnet:?xt=urn:btih:abc123def456..."
```

### Node.js

```js
import { addMagnet } from "@risuko/risuko-js";

const gid = await addMagnet("magnet:?xt=urn:btih:abc123def456...", {
  dir: "/downloads",
});
```

## .torrent Files

### CLI

```bash
risuko download ./ubuntu-24.04.torrent -d ~/Downloads
```

### Node.js

```js
import { readFile } from "node:fs/promises";
import { addTorrent } from "@risuko/risuko-js";

const data = await readFile("./ubuntu-24.04.torrent");
const gid = await addTorrent(data, { dir: "/downloads" });
```

## Seeding Control

By default, Risuko does **not** seed after a torrent completes (`keep-seeding` is `false` and `seed-time` / `seed-ratio` default to `0`). To enable seeding, turn on `keep-seeding`, or set a finite `seed-time` / `seed-ratio`:

### Keep Seeding

Seed indefinitely until the task is stopped manually:

```js
await changeGlobalOption({ "keep-seeding": "true" });
```

When `keep-seeding` is enabled, the `seed-time` and `seed-ratio` limits are ignored.

### Seed Ratio

Stop seeding when the upload/download ratio reaches a threshold:

```bash
risuko download "magnet:?xt=..." --seed-ratio 1.0
```

```js
await addMagnet("magnet:?xt=...", { "seed-ratio": "1.0" });
```

A ratio of `1.0` means upload the same amount as downloaded.

### Seed Time

Seed for a number of minutes after download completes:

```bash
risuko download "magnet:?xt=..." --seed-time 60
```

```js
await addMagnet("magnet:?xt=...", { "seed-time": "60" });
```

<Callout type="info">
Seeding starts whenever `keep-seeding` is true, or `seed-time > 0`, or `seed-ratio > 0`. If `keep-seeding` is off, seeding stops when `seed-time` elapses or `seed-ratio` is reached, whichever comes first.
</Callout>

### Combined Limits

When both `seed-time` and `seed-ratio` are set (and `keep-seeding` is off), seeding stops when **either** limit is reached first.

## Events

BitTorrent downloads emit two completion events:

1. **`risuko.onBtDownloadComplete`** — Download finished, now seeding
2. **`risuko.onDownloadComplete`** — Seeding finished (or limits reached)

```js
onEvent((event, gid) => {
  if (event === "risuko.onBtDownloadComplete") {
    console.log("Download done, seeding...");
  }
  if (event === "risuko.onDownloadComplete") {
    console.log("Fully complete");
  }
});
```

## Peer Information

Check connected peers for a torrent download:

```bash
risuko peers <gid>
```

```js
const peers = await getPeers(gid);
```
